## 1. Record Structure

```jsonc
{
  "id": "kb_20260423_143012_a1b2",        // auto-generated
  "created_at": "2026-04-23T14:30:12+09:00",
  "updated_at": "2026-04-23T14:30:12+09:00",
  "category": "error_resolution",          // required; enumerated
  "title": "PyTorch DDP NCCL timeout on bond0",  // required; 8–60 chars
  "summary": "On 2-node DDP setup...",     // required; 2–4 sentences
  "environment": { /* §2.5 */ },
  "model_config": { /* §2.6; only when relevant */ },
  "error_symptoms": { /* §2.7; required when category=error_resolution */ },
  "solution_steps": [ /* §2.8 */ ],
  "outcome": "success",                    // success | failure | partial
  "confidence": 0.9,                       // 0.0–1.0
  "tags": ["nccl", "ddp", "networking"],
  "related_ids": [],                       // references to other records
  "source": { /* §2.12 */ }
}
```

## 2. Field Reference

### 2.1 `id` (string, auto)
Format: `kb_YYYYMMDD_HHMMSS_<4-char hex hash>`  
Generated by the script. Manual override via `--id` is possible but discouraged.

### 2.2 `created_at` / `updated_at` (string, ISO 8601)
Stored in JST (+09:00). Timezone offset is required.  
On update, only `updated_at` changes.

### 2.3 `category` (enum, required)
See §3.

### 2.4 `title` (string, required)
8–60 characters. Must be self-explanatory in a list view.

- ✗ Bad: `"Error fix"`
- ✔ Good: `"PyTorch DDP NCCL timeout on bond0 interface"`

### 2.5 `environment` (object)

Include only fields relevant to reproduction. Omit rather than leave empty.

```jsonc
{
  "os": "Ubuntu 22.04",
  "kernel": "5.15.0-102",
  "python": "3.11.5",
  "cuda": "12.1",
  "driver": "535.129.03",
  "gpu": "A100 80GB x 8",
  "node_count": 2,
  "interconnect": "InfiniBand HDR",
  "frameworks": {
    "pytorch": "2.3.0",
    "transformers": "4.40.0",
    "vllm": "0.4.2"
  },
  "custom": {}
}
```

### 2.6 `model_config` (object, optional)

Include only when model-specific settings are verified or critical.

```jsonc
{
  "model_name": "Llama-3-70B-Instruct",
  "quantization": "AWQ 4bit",
  "context_length": 8192,
  "tensor_parallel": 8,
  "pipeline_parallel": 1,
  "parameters": {
    "temperature": 0.7,
    "top_p": 0.9,
    "max_new_tokens": 2048
  }
}
```

### 2.7 `error_symptoms` (object, required when `category=error_resolution`)

```jsonc
{
  "error_type": "RuntimeError",
  "error_message": "NCCL timeout after 1800000ms (RANK 7)",
  "stack_trace_excerpt": "...",       // first + last ~50 lines recommended
  "reproduction": [
    "torchrun --nproc_per_node=8 ...",
    "RANK 7 hangs after 30 minutes"
  ],
  "frequency": "always"              // always | intermittent | once
}
```

### 2.8 `solution_steps` (array)

```jsonc
[
  {
    "order": 1,
    "action": "export NCCL_SOCKET_IFNAME=eth0",
    "rationale": "bond0 causes MTU mismatch and high latency",
    "verification": "nccl-tests/all_reduce_perf completes successfully"
  },
  { "order": 2, "action": "...", "rationale": "..." }
]
```

- `order`: 1-based ascending
- `rationale`: **required** — a step without reasoning is a memo, not knowledge
- `verification`: optional but improves reproducibility

### 2.9 `outcome` (enum, required)
- `success`: working state confirmed
- `failure`: tried and did not work (negative knowledge)
- `partial`: partial resolution or temporary workaround

### 2.10 `confidence` (float 0.0–1.0, required)

| Range | Meaning |
|---|---|
| 0.9–1.0 | User explicitly confirmed working |
| 0.6–0.8 | Observed working, no independent verification |
| 0.3–0.5 | Logically sound but untested |
| ≤ 0.2 | Hypothesis or estimate only |

### 2.11 `tags` / `related_ids`
- `tags`: choose from §4 vocabulary (snake_case; custom tags allowed)
- `related_ids`: array of record IDs — use for "precursor problem", "derived issue", etc.

### 2.12 `source` (object, optional)

```jsonc
{
  "conversation_id": "hermes-sess-abc123",
  "agent_version": "hermes-agent v0.3.2",
  "user_hint": "section the user explicitly flagged as important"
}
```

## 3. Category Taxonomy

| Category | Knowledge type |
|---|---|
| `error_resolution` | Error symptom + resolution pair. `error_symptoms` field is required |
| `environment_setup` | Verified setup procedure, pinned dependencies |
| `model_config` | Recommended settings or hyperparameters for a specific model |
| `workflow` | Multi-step reusable procedure |
| `best_practice` | Principles or anti-patterns that generalize beyond a single problem |

When multiple categories apply, choose the one most useful as a **future search entry point**. Example: an OOM error encountered during inference server setup is better stored as `error_resolution` (you'll search by error next time) than `environment_setup`.

## 4. Tag Vocabulary

Use these tags by default. New snake_case tags may be added as needed.

**Infrastructure:** `cuda`, `rocm`, `cpu`, `gpu`, `networking`, `storage`, `docker`, `k8s`, `systemd`

**Distributed/Parallel:** `ddp`, `fsdp`, `deepspeed`, `tensor_parallel`, `pipeline_parallel`, `nccl`

**Error types:** `oom`, `timeout`, `dependency_conflict`, `version_mismatch`, `permission`, `segfault`, `hang`

**Model-related:** `inference`, `training`, `finetune`, `quantization`, `rag`, `agent`, `tool_use`

**Languages/Frameworks:** `python`, `pytorch`, `jax`, `transformers`, `vllm`, `sglang`, `ray`, `flash_attention`

## 6. Storage Adapters

The scripts select a backend via the `HERMES_KB_BACKEND` environment variable. When `HERMES_KB_CLI_SEARCH` is set, the scripts auto-select `cli` without needing to set `HERMES_KB_BACKEND` explicitly.

### 6.1 `cli` — LLM-Wiki (Recommended Default)

Provide command templates via environment variables. `{query}`, `{limit}`, `{json}` are placeholders.

```bash
export HERMES_KB_BACKEND=cli
export HERMES_KB_CLI_SEARCH='llm-wiki search --query {query} --limit {limit} --format json'
export HERMES_KB_CLI_UPDATE='llm-wiki add --from-stdin'
```

- The search command must write a **JSON array** to stdout (each element is a record conforming to this schema)
- The update command reads JSON from stdin and exits with code 0 on success
- Timeout: 30 s (hardcoded in scripts)

### 6.2 `jsonl` — Local fallback

- Store: `$HERMES_KB_DIR/kb.jsonl` (one JSON record per line)
- Index: `$HERMES_KB_DIR/index.md` (auto-regenerated on every write — **do not edit manually**)
- Tag index: `$HERMES_KB_DIR/tags.idx.json` (optional; activated by `--use-index`)
- Default directory when `HERMES_KB_DIR` is unset: `~/.hermes-agent/kb/`
- Writes use `kb.jsonl.tmp` → atomic rename to prevent partial writes

### 6.3 `http` — REST API

```bash
export HERMES_KB_BACKEND=http
export HERMES_KB_HTTP_ENDPOINT=http://memory-server:8080
# Expected endpoints:
#   POST /search  body: {"query":"...","category":"...","tags":[...],"limit":5}
#   POST /upsert  body: <Record>
#   GET  /record/<id>
```

Authentication: set `HERMES_KB_HTTP_TOKEN`; scripts send `Authorization: Bearer <token>`.  
Timeout default: 10 s; override with `HERMES_KB_HTTP_TIMEOUT`.

### 6.4 Fallback Rules

On `cli` or `http` failure, scripts automatically fall back to `jsonl` and emit a warning to stderr. Pass `--no-fallback` to disable.

## 7. Canonical Examples

### Example 1 — Error Resolution (distributed training)

```json
{
  "category": "error_resolution",
  "title": "PyTorch DDP NCCL timeout on bond0",
  "summary": "On 2-node DDP (8× A100 per node), NCCL timed out when NCCL_SOCKET_IFNAME was unset because bond0 caused MTU mismatch. Explicitly setting eth0 resolved it. Single env-var change.",
  "environment": {
    "os": "Ubuntu 22.04",
    "cuda": "12.1",
    "gpu": "A100 80GB x 8",
    "node_count": 2,
    "interconnect": "InfiniBand HDR",
    "frameworks": { "pytorch": "2.3.0" }
  },
  "error_symptoms": {
    "error_type": "RuntimeError",
    "error_message": "NCCL timeout after 1800000ms RANK 7",
    "reproduction": ["torchrun --nnodes=2 --nproc_per_node=8 train.py"],
    "frequency": "always"
  },
  "solution_steps": [
    {
      "order": 1,
      "action": "export NCCL_SOCKET_IFNAME=eth0",
      "rationale": "bond0 causes MTU mismatch and high latency",
      "verification": "all_reduce_perf completes without timeout"
    },
    { "order": 2, "action": "relaunch torchrun", "rationale": "apply the env var" }
  ],
  "outcome": "success",
  "confidence": 0.95,
  "tags": ["nccl", "ddp", "networking", "timeout"]
}
```

## 8. Index Markdown Format

`search_knowledge.py --list-index --format markdown` outputs the following format (identical to auto-generated `$HERMES_KB_DIR/index.md`).

**Do not edit `index.md` directly** — it is regenerated by `update_knowledge.py` on every write.

```markdown
# Hermes-Agent Knowledge Base Index

_Generated: 2026-04-23 14:30 JST / Total records: 42_

## error_resolution (18)

- `kb_20260423_143012_a1b2` **PyTorch DDP NCCL timeout on bond0** — `nccl` `ddp` `networking` — success (0.95)
- `kb_20260422_091530_c3d4` **FlashAttention-2 incompatible with PyTorch 2.0** — `flash_attention` `version_mismatch` — success (0.85)

## environment_setup (12)

- `kb_20260420_180000_e5f6` **vLLM 0.4.2 Llama-3-70B on 4× A100** — `vllm` `inference` — success (0.9)

## model_config (7) / workflow (3) / best_practice (2)
```
